Introduction to HTML Forms

HTML forms are used to collect user input on a webpage. The information entered by users (such as names, emails, or passwords) is usually sent to a server for processing.

Forms are commonly used for:

  • Log in pages
  • Registration Forms
  • Surveys
  • Search Boxes

An HTML form is created using the [form] element, which acts as a container for different input elements like text fields, checkboxes, radio buttons, and submit buttons.

The [form] Element

The [form] tag defines an HTML form where users can enter and submit data. It groups together all the input controls that collect information.

When the user submits the form, the browser sends the data to a server or another webpage for processing.

Common HTML Form Elements

Forms contain different elements that allow users to enter or select information.

[input] - The most commonly used form element. It creates different input controls depending on the type attribute.

Value Description
text single-line text field
radio select one option from many
chechkbox select multiple options
submit button that submits the form
button clickable button

[label] - Defines a label for an input field and improves accessibility by helping screen readers identify the input.


[select] and [option] - Used to create dropdown lists where users can select one option.


[textarea] - Creates a multi-line text input field.


[button] - Creates a clickable button that can trigger actions or submit the form.


[fieldset] and [legend]

  • [fieldset] - groups related form elements
  • [legend] - provides a title or caption for the group

[datalist] and [output]

  • [datalist] - provides predefined suggestions for input fields
  • [output] - displays the result of calculations or scripts

Important Form Attributes

  • text/plain
    • Attribute Value Description
    action url
    • An absolute URL - points to another web site (like action="http://www.example.com/example.htm")
    • A relative URL - points to a file within a web site (like action="example.htm")
    		 Specifies where the form data will be sent after submission. If not specified, the data is sent to the current page.
    method
    • get - sends data through the URL
    • post - sends data securely in the request body
    		 Defines the HTTP method used to send data
    target
    • _blank - The response is displayed in a new window or tab
    • _self - The response is displayed in the current window (default)
    • _parent - The response is displayed in the parent frame
    • _top - The response is displayed in the full body of the window
    • framename - The response is displayed in a named iframe
    		 Specifies where the response should appear after the form is submitted, such as:
    autocomplete
    • on
    • off
    		 Determines whether the browser should automatically fill in previously entered values.
    novalidate boolean attribute (its mere presence indicates true, while its absence signifies false) Specifies that the form should not validate input data before submission.
    enctype
    • application/x-www-form-urlencoded
    • multipart/form-data
    		 Defines how the form data should be encoded when it is sent to the server, especially when uploading files.

    How Forms Work

    The general process of an HTML form is:

    • The user enters information in the form fields.
    • The user clicks the submit button.
    • The browser sends the data to the server using the method specified in the form.
    • The server processes the data and returns a response.
